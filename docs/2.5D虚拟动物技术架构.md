# Pet2.0 - 2.5D虚拟宠物技术架构文档

## 1. 架构概述

基于 HarmonyOS NEXT (API 12+) 官方推荐和 `DengShiyingA/harmonyos-ai-skill` 性能最佳实践，本次重构抛弃原有的 `ArkGraphics3D` (`SceneView` + `.glb`模型)，彻底转向基于 ArkUI 原生渲染树的 **2.5D 分层交互引擎架构**。该架构以极低的性能开销（免初始化 OpenGL/Vulkan 上下文），利用属性动画与多图层合成实现立体效果。

## 2. 核心技术栈与选型

1. **框架语言**：ArkTS (Stage 模型, 严格模式, 禁止使用 `any`)
2. **渲染方案**：
   * **动态骨骼方案**：`@ohos/lottie`，利用 AE 导出的 JSON 实现高帧率 2.5D 骨骼形变（无需三方 C++ 渲染库）。
   * **序列帧备用方案**：ArkUI 原生 `ImageAnimator`，适用于高保真预渲染 3D 素材的降级展示。
3. **空间合成**：ArkUI 原生 `Stack` 布局结合矩阵变换（`.rotate`, `.translate`, `.scale`）。
4. **手势系统**：ArkUI `GestureGroup` (包含 `PanGesture`, `PinchGesture`, `TapGesture`)。

## 3. 分层视差渲染模型 (Parallax Render Tree)

在 `PetSprite.ets` 组件中，使用 `Stack` 从下至上严格按照层级关系渲染：

```arkts
Stack({ alignContent: Alignment.Center }) {
  // Layer 1: 环境阴影 (Environment Shadow)
  // 通过透明度和缩放模拟宠物与地面的距离感
  Ellipse().fill('#000000').opacity(0.2)

  // Layer 2: 宠物本体 (Pet Main Body - Lottie / ImageAnimator)
  // 绑定 @State petRotationY, petScale 等手势偏移数据
  LottieComponent({ path: this.pet.lottiePath })
    .rotate({ x: this.petRotationX, y: this.petRotationY, z: 0, angle: 1 })
    .scale({ x: this.petScale, y: this.petScale })

  // Layer 3: 前景交互元素 (Foreground UI)
  // 悬浮气泡或粒子装饰，位移速率高于 Layer 2，形成景深
  BubbleText({ text: this.pet.currentSpeech })
    .translate({ x: this.bubbleOffsetX, y: this.bubbleOffsetY })
}
```

## 4. 状态管理与数据流

采用鸿蒙 V1/V2 装饰器体系，实现单向数据流与细粒度更新：

*   **全局数据模型**：`PetModel.ets`，将 `modelPath` 替换为 `lottiePath: string`。
*   **组件级状态 (`@State`)**：
    *   `@State petRotationX/Y`: 捕捉手指滑动的偏移，利用 `Math.min/max` 进行防穿帮角度钳制（±30°）。
    *   `@State petScale`: 捕获捏合手势的系数。
    *   `@State currentAnim`: 记录 Lottie 当前播放的动画剪辑名（如 `'idle'`, `'eating'`）。
*   **父子通信 (`@ObjectLink` / `@Prop`)**：
    *   在 `Index.ets` 获取 `@StorageLink('currentPet')`，并通过 `@ObjectLink` 下发给 `PetSprite` 组件，确保内存地址变动时局部刷新。

## 5. 性能优化策略 (Performance Best Practices)

根据 `Awesome HarmonyOS NEXT` 与 AI Skill 指南，强制遵守以下优化规则：

1. **动画曲线 (Animation Curves)**：所有的交互恢复（如松手后模型回正）必须使用 `Curve.SpringMotion` 或 `Curve.EaseOut` 替代生硬的赋值，开启系统的帧插值能力。
2. **手势节流 (Throttling)**：在 `.onActionUpdate` 频繁触发时，ArkUI 默认会进行合并绘制，但建议通过简单系数缩小偏移量，避免渲染跳帧。
3. **渲染优化**：使用 `.renderGroup(true)` 隔离宠物动画层与外部静态 UI，防止局部高频重绘引发整树 Relayout。
4. **资源懒加载**：Lottie 动画 JSON 文件通过异步 `lottie.loadAnimation()` 载入，不阻塞主线程。

## 6. 代码审查关注点 (HarmonyOS Review Guidelines)

在将来的开发迭代中，基于 `harmonyos-review` 技能标准：
*   **组件生命周期**：在 `aboutToDisappear` 时，必须调用 `lottie.destroy()` 释放 C++ 层面的动画缓存，防止内存泄漏。
*   **UI 规范**：背景色统一为 `$r('sys.color.ohos_id_color_sub_background')` 或设计稿深色(`#081e2a`)，阴影统一使用 `shadow({ radius: 16 })`。
