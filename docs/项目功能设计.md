# 虚拟数字宠物 — 项目功能设计

> 项目名称：**鸿蒙虚拟数字宠物（HarmoPet）**  
> 选题方向：电子萌宠（虚拟数字宠物）  
> 开发策略：**软件为主，硬件待定** — 模拟器(Previewer)+手机为核心运行平台，完整覆盖全部功能；SC-3568HA全息端 + Hi3863南向端为可选增强（硬件待定，视获取情况决定是否集成）  
> 设计理念：**有灵魂的数字伙伴** — 不只是动画，而是拥有记忆、情绪与成长的生命

---

## 一、产品定位与核心理念

### 1.1 产品定位

一款基于**开源鸿蒙系统（OpenHarmony / SwanLinkOS）**的**虚拟数字宠物**，融合AI智能与IoT感知，打造能**永久记忆互动内容、支持语音交流、具备情绪共鸣**的数字陪伴伙伴。

### 1.2 核心理念

| 理念 | 说明 |
|------|------|
| 🧠 有记忆 | 三层记忆架构，记住每一次互动、每一个偏好 |
| 😊 有情绪 | 感知用户情绪并共鸣，宠物自身也有喜怒哀乐 |
| 🌱 有成长 | 性格随互动演变，从陌生到亲密的成长轨迹 |
| 🗣️ 能对话 | 自然语音交互，像真实伙伴一样聊天 |
| 🔗 能感知 | 南向传感器感知环境，宠物对真实世界有反应 |
| 💭 会做梦 | 睡眠时生成梦境，拥有自己的内心世界 |

### 1.3 与赛题的对标关系

> **我们的选题方向：电子萌宠**（赛题给出的5个参考方向之一）
> 
> 赛题说"以下为一些可供参考的应用场景"，我们选择"电子萌宠"作为核心方向。
> 由于虚拟电子宠物天然涉及养宠的多个维度，我们的实现在聚焦电子萌宠的同时，
> 自然覆盖了其他场景的部分能力，但这些不是独立模块，而是电子萌宠的有机组成。

| 赛题场景 | 与我们的关系 | 说明 |
|----------|-------------|------|
| **电子萌宠** | **🎯 核心选题** | 虚拟数字宠物，永久存储互动、语音交流、情绪共鸣 — 这是我们的全部重心 |
| 宠物健康管理 | 顺带覆盖 | 电子宠物有6维属性（含健康值），健康低了会生病，这是宠物自身的玩法机制，不是独立的"健康管理模块" |
| 宠物行为监测 | 顺带覆盖 | 南向传感器让宠物能感知环境，这是宠物的"感知能力"，不是独立的"行为监测系统" |
| 智能互动玩具 | 顺带覆盖 | 语音/触摸互动是电子萌宠的核心交互方式，不是独立的"玩具产品" |
| 宠物档案 | 顺带覆盖 | 宠物成长记录是记忆系统的一部分，不是独立的"档案管理系统" |
| 宠物领养系统 | 顺带覆盖 | 开局领养流程是电子萌宠的入门体验，不是独立的"领养平台" |

### 3.2 六维属性系统

| 属性 | 范围 | 影响因素 | 影响 |
|------|------|----------|------|
| ❤️ 健康 | 0-100 | 饮食规律、环境舒适度 | 宠物活力、生病概率 |
| 😊 心情 | 0-100 | 互动频率、用户情绪共鸣 | 行为表现、对话语气 |
| ⚡ 精力 | 0-100 | 互动消耗、睡眠恢复 | 可执行动作种类 |
| 💕 亲密度 | 0-100 | 互动次数、记忆深度 | 宠物主动行为频率 |
| 🐾 顺从度 | 0-100 | 训练、奖惩机制 | 宠物服从指令概率 |
| 😏 毒舌值 | 0-100 | 互动方式、性格演化 | 对话风格（温柔↔毒舌） |

**属性变化规则：**
- 每10分钟自然衰减1-3点（健康、心情、精力）
- 互动行为恢复对应属性5-15点
- 连续不互动超24小时，属性加速衰减
- 亲密度只增不减（记录累计互动）

## 四、AI智能模块

### 4.1 AI对话系统

#### 对话能力

| 能力 | 说明 |
|------|------|
| 自然对话 | 接入LLM，以宠物视角进行日常聊天 |
| 上下文理解 | 短期记忆保持对话连贯性 |
| 个性化表达 | 根据性格维度调整语气（温柔/毒舌/活泼） |
| 情绪回应 | 感知用户情绪后给出适当回应 |
| 主动关心 | 宠物主动询问用户状态（基于记忆触发） |
| 知识问答 | 简单百科问答（宠物视角回答） |

#### 对话人格设定（Prompt模板）

```
你是{pet_name}，一只{pet_breed}虚拟宠物。
你的性格：开朗度{cheerful}/100，调皮度{naughty}/100...

记住以下关于主人的信息：
{user_profile_summary}

最近的互动记忆：
{recent_memories}

请以宠物的视角回复主人，保持角色一致性。
回复要简洁可爱，适当使用语气词。
```

### 4.2 语音交互系统

#### 交互流程

```

│ 按住   │    │ VAD   │    │Whisper│    │ LLM   │    │Edge   │
│ 说话   │───→│端点   │───→│语音   │───→│生成   │───→│TTS    │

└───────┘    └───────┘    └───────┘    └───────┘    └───────┘
```

#### 语音特性

| 特性 | 方案 |
|------|------|
| 语音识别 | Whisper本地离线识别，支持中英文 |
| 语音合成 | Edge-TTS在线合成，多种音色可选 |
| 声音风格 | 根据宠物类型选择（猫→软萌、狗→活力） |
| VAD | 语音活动检测，自动判断说话结束 |
| 语音缓存 | 高频回复预合成缓存，减少延迟 |

### 4.3 记忆系统

> ⚠️ **赛题合规说明**：赛题要求"能够永久存储互动内容"。记忆系统采用**北向本地RDB + AI后端双层持久化**，确保互动内容永不丢失。

#### 记忆类型

| 记忆类型 | 北向本地存储 | AI后端存储 | 保留时长 | 示例 |
|----------|------------|-----------|----------|------|
| 对话记忆 | RDB: chat_history | 内存队列 | 当前会话+永久归档 | "主人刚才说了想去旅游" |
| 语义记忆 | RDB: interaction_log | ChromaDB向量库 | 永久（可检索） | "主人喜欢猫不喜欢狗" |
| 长期记忆 | RDB: pet_growth | SQLite | 永久 | "主人的生日是3月15日" |
| 情感记忆 | RDB: emotion_record | 关联存储 | 永久 | "上次主人哭的时候我安慰了ta" |
| 行为记忆 | RDB: interaction_log | 统计聚合 | 永久 | "主人每天晚上10点会来" |

#### 记忆操作

| 操作 | 触发 | 说明 |
|------|------|------|
| 存入 | 每次对话结束 | 关键信息提取→分类存储 |
| 检索 | 每次对话开始 | 语义相似度检索相关记忆 |
| 压缩 | 每10轮对话 | 短期记忆摘要→语义记忆 |
| 遗忘 | 超过容量/低相关性 | 弱关联记忆降权或删除 |
| 回忆 | 用户询问/主动触发 | 检索并讲述相关记忆 |

#### 用户画像自动构建

```
用户画像 = {
    基本信息: { 昵称, 生日, 职业 },
    偏好: { 喜欢的食物, 宠物偏好, 作息规律 },
    情绪模式: { 压力时段, 开心触发, 安慰方式 },
    互动习惯: { 活跃时段, 互动频率, 偏好互动类型 },
    重要事件: [ "2026-04-01: 主人考试通过了" ]
}
```

### 4.5 做梦系统

#### 做梦流程

```

    ▼

    ▼
```

#### 梦境类型

| 梦境类型 | 触发条件 | 内容示例 |

| 美食梦 | 饥饿状态入睡 | 梦到吃不完的小鱼干 |

宠物醒来后，可以主动告诉主人梦境内容，增加情感连接：
- "主人，我昨晚做了一个好梦，梦到我们一起去了海边~"
- "呜呜，我做了噩梦，梦到你不要我了..."

## 五、IoT感知模块（南向）

> 📡 **南向设备角色**：Hi3863星闪开发板作为环境感知与交互反馈终端，通过星闪通信与北向应用协同，为宠物赋予对真实世界的感知能力。

### 5.1 硬件平台

#### 开发板配置

```
Hi3863星闪开发板（OpenHarmony轻量系统）
├── 主控芯片：Hi3863V100（ARM Cortex-M4，160MHz）
├── 通信模块：星闪（SparkLink）无线通信
├── 传感器接口：E53接口标准
├── 电源管理：USB供电/电池供电
└── 存储资源：352KB SRAM + 288KB ROM
```

#### E53传感器模块（智能宠物扩展板）

| 模块 | 型号 | 功能 | 对宠物系统的作用 |
|------|------|------|------------------|
| 温湿度传感器 | AHT20 | 环境温湿度检测 | 环境舒适度计算 |
| 人体红外传感器 | AS312 | 检测用户靠近/离开 | 触发欢迎/等待行为 |
| 三轴加速度计 | LIS3DH | 检测设备振动/晃动 | 互动反馈、活动量统计 |
| 光照传感器 | AP3216C | 检测环境光照 | 昼夜判断、睡眠触发 |
| OLED显示屏 | SSD1306 | 0.96寸屏幕显示 | 南向端显示宠物状态 |
| LED灯 | WS2812 | RGB彩灯 | 表情/状态可视化 |
| 蜂鸣器 | 无源蜂鸣器 | 发声反馈 | 宠物声音模拟 |
| 振动马达 | 1027震动马达 | 振动反馈 | 互动触感 |

### 5.2 环境感知系统

#### 环境数据采集

| 采集项 | 传感器 | 采样频率 | 数据格式 |
|--------|--------|----------|----------|
| 温度 | AHT20 | 1Hz | `{temp: 26.5}` °C |
| 湿度 | AHT20 | 1Hz | `{humidity: 58}` % |
| 光照强度 | AP3216C | 0.5Hz | `{light: 250}` lux |
| 设备温度 | 板载传感器 | 0.2Hz | `{board_temp: 32.1}` °C |

#### 环境数据映射规则

| 环境状态 | 判断条件 | 影响宠物属性 |
|----------|----------|--------------|
| 🔥 高温 | 温度 > 35°C | 心情-5/小时，健康-3/小时 |
| ❄️ 低温 | 温度 < 10°C | 心情-5/小时，健康-3/小时 |
| 💧 潮湿 | 湿度 > 80% | 心情-3/小时，健康-2/小时 |
| 🌵 干燥 | 湿度 < 30% | 心情-3/小时，健康-2/小时 |
| 🌞 强光 | 光照 > 1000 lux | 无影响（白天模式） |
| 🌙 弱光 | 光照 < 100 lux | 触发睡眠状态 |
| ✅ 舒适 | 温度20-28°C，湿度40-70% | 心情+5/小时 |

#### 舒适度计算模型

```
舒适度指数 = 100 - |温度-25|×2 - |湿度-55|×0.5

分级标准：
├── 优秀 (80-100)：宠物非常舒适，心情自然恢复
├── 良好 (60-80)：宠物感觉一般，无影响
├── 较差 (40-60)：宠物不适，心情缓慢下降
└── 恶劣 (0-40)：宠物很难受，心情&健康快速下降

应用场景：
├── 环境数据变化 → 实时计算舒适度 → 上报北向
├── 舒适度<60 → 触发环境异常事件
└── 舒适度变化记录 → 存入宠物成长日志
```

### 5.3 活动检测系统

#### 检测项目与实现

| 检测项 | 传感器 | 检测算法 | 上报条件 |
|--------|--------|----------|----------|
| 用户靠近 | AS312人体红外 | 电平触发 | 检测到人体→立即上报 |
| 用户离开 | AS312人体红外 | 超时判断 | 无人体信号>30秒→上报 |
| 设备晃动 | LIS3DH加速度计 | 加速度突变检测 | 加速度>阈值→上报 |
| 设备静止 | LIS3DH加速度计 | 长时间无振动 | 静止>10分钟→上报 |
| 环境振动 | LIS3DH加速度计 | 持续振动检测 | 振动强度>设定值→上报 |

#### 活动检测流程

```
传感器数据采集 (10Hz)
    │
    ├── 人体红外检测
    │   ├── 高电平 → 用户靠近事件 → 星闪上报
    │   └── 低电平持续30秒 → 用户离开事件 → 星闪上报
    │
    └── 加速度计检测
        ├── 突变检测 (窗口滑动方差算法)
        │   ├── 方差>阈值 → 晃动事件 → 星闪上报
        │   └── 方差<阈值 → 正常状态
        └── 长时静止检测
            └── 无振动>10分钟 → 静止事件 → 星闪上报
```

#### 活动量统计

```
活动量计算 = Σ|加速度变化| × 时间权重

用途：
├── 统计用户与设备互动的频率和强度
├── 映射到宠物的"运动量"属性
├── 影响宠物的精力消耗和心情变化
└── 生成每日活动报告 → 上报北向记录
```

```
└─────────────┘                          └─────────────┘
     │                                          │
     ├─ 传感器数据上报                           ├─ 数据处理与存储
     ├─ 事件通知                                ├─ 宠物状态更新
     └─ 心跳保活                                └─ 控制指令下发
```

#### 星闪通信协议

**数据上报格式：**

```json
{
  "msg_type": "sensor_data",
  "timestamp": 1681234567890,
  "data": {
    "temp": 26.5,
    "humidity": 58,
    "light": 250,
    "accel_x": 0.02,
    "accel_y": 0.01,
    "accel_z": 0.98
  }
}
```

**事件上报格式：**

```json
{
  "msg_type": "event",
  "timestamp": 1681234567890,
  "event": {
    "type": "user_approach",  // user_leave / shake / stillness
    "data": {
      "duration": 5  // 秒
    }
  }
}
```

**控制指令格式：**

```json
{
  "msg_type": "command",
  "cmd": "buzzer",
  "params": {
    "freq": 2000,
    "duration": 500
  }
}
```

#### 心跳机制

```
心跳频率：10秒/次
超时判定：连续3次未响应(30秒) → 设备离线
重连机制：离线后每5秒尝试重连
```
