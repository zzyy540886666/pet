# 核心宠物模块 — 模块设计文档

> **模块名称**: 核心宠物模块 (Core Pet Module)  
> **所属项目**: 鸿蒙虚拟数字宠物 (HarmoPet)  
> **模块版本**: v1.0  
> **更新日期**: 2026-04-15

---

## 1. 模块概述

### 1.1 模块定位

核心宠物模块是 HarmoPet 的**基础与核心**，负责宠物的视觉呈现、属性管理、情绪表达、性格演化以及用户互动行为处理。该模块独立于 AI 服务和 IoT 设备运行，确保在离线状态下仍可提供完整的宠物养成体验。

### 1.2 模块职责

| 职责 | 说明 |
|------|------|
| 宠物精灵渲染 | 基于 Canvas 帧动画渲染宠物形象，支持多状态动作切换 |
| 属性系统管理 | 维护六维属性（健康/心情/精力/亲密度/顺从度/毒舌值） |
| 情绪引擎 | 基于多源输入计算并驱动宠物情绪状态变化 |
| 性格成长 | 记录性格维度演化，支持四阶段成长轨迹 |
| 互动行为 | 处理用户触摸/喂食/玩耍等交互操作 |

### 1.3 依赖关系

```
核心宠物模块
├── 依赖：北向数据层 (RDB / PersistentStorage)
├── 被依赖：AI智能模块（对话人格注入宠物状态）
├── 被依赖：IoT感知模块（环境事件影响属性/情绪）
└── 被依赖：全息双端模块（同步宠物动画/状态）
```

## 2. 宠物精灵子系统

### 2.1 功能描述

宠物精灵是用户的视觉交互中心，以卡通风格的虚拟宠物形象展示在主界面，通过帧动画表现不同动作和情绪。

#### 外观设计规范

| 要素 | 规范 |
|------|------|
| 造型 | 卡通风格，支持小猫/小狗/兔子三种品种 |
| 渲染方式 | Canvas 帧动画 + SpriteSheet 精灵图 |
| 显示尺寸 | 主界面居中，约占屏幕 1/3 区域 |
| 视觉风格 | 扁平化 + 微立体，可爱治愈风 |

#### 动作库定义

**MVP 最小动作集（必须实现）：**

| 动作ID | 动作名称 | 分类 | 触发条件 | MVP优先级 |
|--------|----------|------|----------|-----------|
| `idle` | 站立发呆 | 闲置 | 无交互5秒后随机触发 | P0 |
| `look_around` | 左右张望 | 闲置 | 闲置状态循环播放 | P0 |
| `yawn` | 打哈欠 | 闲置 | 闲置>10秒或精力<30 | P0 |
| `happy_spin` | 开心转圈 | 情绪 | 心情>80时触发 | P0 |
| `eat` | 吃东西 | 日常 | 用户喂食时 | P0 |
| `sleep` | 睡觉 | 日常 | 精力<20或夜间模式 | P0 |
| `pet_enjoy` | 享受抚摸 | 互动 | 用户触摸滑动 | P0 |
| `jump` | 兴奋跳跃 | 移动 | 用户靠近/开心时 | P1 |
| `sad` | 委屈蹲下 | 情绪 | 心情<30时 | P1 |
| `angry` | 生气跺脚 | 情绪 | 毒舌值高+被频繁打扰 | P2 |

    ┌────┴────┐  ┌──────────┐  ┌───────┐

## 3. 属性子系统

### 3.1 功能描述

维护宠物的六维属性数值，负责属性的自然衰减、互动恢复、阈值判定及持久化存储。属性系统是宠物"活着"的核心机制——属性变化直接影响宠物行为和情绪。

#### 六维属性定义

| 属性ID | 名称 | 范围 | 初始值 | 影响因素 | 影响效果 |
|--------|------|------|--------|----------|----------|
| `health` | 健康 | 0-100 | 80 | 饮食规律、环境舒适度 | 活力、生病概率 |
| `mood` | 心情 | 0-100 | 70 | 互动频率、情绪共鸣 | 行为表现、对话语气 |
| `energy` | 精力 | 0-100 | 90 | 互动消耗、睡眠恢复 | 可执行动作种类 |
| `intimacy` | 亲密度 | 0-100 | 0 | 互动次数、记忆深度 | 宠物主动行为频率 |
| `obedience` | 顺从度 | 0-100 | 50 | 训练、奖惩机制 | 服从指令概率 |
| `snarkiness` | 毒舌值 | 0-100 | 30 | 互动方式、性格演化 | 对话风格（温柔↔毒舌） |

#### MVP 属性变化规则

```typescript
// 属性自然衰减配置（每10分钟触发一次）

  intimacy:     { minDecay: 0, maxDecay: 0, intervalMs: 0 },  // 只增不减
  obedience:    { minDecay: 0, maxDecay: 0, intervalMs: 0 },  // 仅训练改变
  snarkiness:   { minDecay: 0, maxDecay: 0, intervalMs: 0 },  // 仅性格演化改变

// 互动恢复配置
const INTERACTION_RECOVERY_CONFIG: Record<InteractionType, Partial<Record<AttributeId, number>>> = {

  pet:     { mood: [8, 12], intimacy: [3, 6] },
  play:    { mood: [8, 12], energy: [-15, -8] },
  train:   { obedience: [3, 7], snarkiness: [-3, 3] },
  bath:    { health: [8, 12], mood: [-5, 5] },

// 特殊规则

## 4. 情绪子系统

### 4.1 功能描述

情绪系统基于多源输入计算宠物当前情绪状态，驱动表情变化和动作选择。MVP版本支持基础情绪模型，完整版本接入AI情绪共鸣引擎。

#### 基础情绪空间

```
情绪 = f(用户情绪, 宠物属性, 环境状态, 记忆影响)

┌───────────────────────────────────┐
│         宠物情绪空间               │
│                                   │
│           😄 快乐                  │
│          ╱    ╲                   │
│    😎兴奋 ────── 😌平静            │
│        ╲    ╱                     │
│    😰焦虑 ────── 😢悲伤            │
│          ╲    ╱                   │
│           😴 困倦                  │
│                                   │
└───────────────────────────────────┘
```

#### MVP 情绪转换规则

| 当前情绪 | 触发条件 | 目标情绪 | 优先级 |
|----------|----------|----------|--------|
| 平静 | 心情 > 75 | 快乐 | 高 |
| 平静 | 精力 < 25 | 困倦 | 高 |
| 快乐 | 长时间无互动 (>30min) | 平静 | 中 |
| 快乐 | 心情 < 35 | 悲伤 | 中 |
| 快乐 | 被频繁打扰 (短时间多次互动) | 焦虑 | 低 |
| 悲伤 | 有安慰互动 | 快乐 | 中 |
| 焦虑 | 环境恢复/安静 | 平静 | 低 |
| 任意 | 精力耗尽 (=0) | 困倦 | 最高 |

### 4.2 接口定义

#### 情绪数据模型

## 5. 性格成长子系统

### 5.1 功能描述

性格成长系统记录宠物五维性格的演化过程，根据用户互动方式逐步塑造独特人格。MVP版本实现基础的性格记录与显示，完整版本支持性格对对话和行为的影响。

#### 五维性格定义

| 维度ID | 名称 | 低位表现 (0-30) | 中位表现 (31-70) | 高位表现 (71-100) | 主要成长方式 |
|--------|------|-----------------|------------------|-------------------|-------------|
| `cheerfulness` | 开朗度 | 安静内敛 | 正常 | 活泼话多 | 多互动、多夸赞 |
| `independence` | 独立性 | 粘人依赖 | 平衡 | 自主独立 | 适当离开、不过度互动 |
| `curiosity` | 好奇心 | 安于现状 | 一般 | 探索欲强 | 提供新场景、小游戏 |
| `naughtiness` | 调皮度 | 乖巧听话 | 偶尔捣蛋 | 恶作剧大师 | 多样化互动方式 |
| `warmth` | 暖心度 | 直来直去 | 温柔 | 体贴关心 | 分享情绪、深度对话 |

#### 成长阶段

| 阶段ID | 阶段名称 | 时间范围 | 性格特征 | 对话能力 |
|--------|----------|----------|----------|----------|
| `stranger` | 初识期 | 第0-7天 | 害羞、简单反应 | 基础问候 |
| `familiar` | 熟悉期 | 第8-30天 | 主动互动初显 | 记忆积累 |
| `intimate` | 亲密期 | 第31-90天 | 深度共鸣 | 情绪理解 |
| `soulmate` | 默契期 | 90天以上 | 独特人格 | 心有灵犀 |

### 5.2 接口定义

#### 性格数据模型

}
```

## 6. 互动行为子系统

### 6.1 功能描述

互动行为子系统处理所有用户与宠物之间的直接交互操作，包括触摸、喂食、玩耍、训练、洗澡、睡觉等。每次互动都会触发对应的属性变化和宠物反馈。

#### 互动类型定义（MVP）

| 互动ID | 操作方式 | 宠物反馈(MVP) | 属性影响 | MVP优先级 |
|--------|----------|---------------|----------|-----------|
| `pet` | 在宠物区域触摸滑动 | 闭眼享受动画 + 呼噜声 | 心情+[8,12] 亲密度+[3,6] | P0 |
| `feed` | 点击食物图标 → 选择食物 | 吃东西动画 + 满足气泡 | 健康+[10,15] 精力+[3,8] | P0 |
| `play` | 选择玩具/触发小游戏 | 兴奋跳跃动画 | 心情+[8,12] 精力-[15,8] | P0 |
| `train` | 选择指令教学 | 学习动作动画 | 顺从度+[3,7] 毒舌±[3] | P1 |
| `bath` | 点击洗澡图标 | 先躲闪后享受动画 | 健康+[8,12] 心情-[5,5] | P1 |
| `sleep` | 点击睡觉/夜间模式 | 打哈欠入睡动画 | 精力→100 | P0 |

### 6.2 接口定义

```typescript
// 文件路径: services/InteractionService.ts
// 注意：此服务同时被UI层和全息同步层调用

export type InteractionType = 'pet' | 'feed' | 'play' | 'train' | 'bath' | 'sleep';

export interface InteractionResult {
  success: boolean;
  petReaction: string;       // 宠物反馈动作ID
  attributeChanges: Partial<Record<AttributeId, number>>;  // 属性变化量
  bubbleText?: string;       // 气泡文本
  timestamp: number;
}

export interface FeedOptions {
  foodId: string;            // 食物ID: 'cat_food' | 'fish' | 'milk' ...
  quantity?: number;         // 数量，默认1
}

export interface PlayOptions {
  toyId: string;             // 玩具ID: 'ball' | 'yarn' | 'frisbee' ...
  duration?: number;         // 游戏时长(ms)，默认30000
}

export class InteractionService {
  private static instance: InteractionService;
  
  public static getInstance(): InteractionService
  
  // ---------- 互动执行 ----------
  /** 抚摸互动 */
  async pet(): Promise<InteractionResult>
  
  /** 喂食互动 */
  async feed(options: FeedOptions): Promise<InteractionResult>
  
  /** 玩耍互动 */
  async play(options: PlayOptions): Promise<InteractionResult>
  
  /** 训练互动 */
  async train(commandId: string): Promise<InteractionResult>
  
  /** 洗澡互动 */
  async bath(): Promise<InteractionResult>
  
  /** 睡觉互动 */
  async sleep(): Promise<InteractionResult>
  
  // ---------- 通用方法 ----------
  /**
   * 记录互动到RDB（永久存储合规）
   */
  private logInteraction(type: InteractionType, result: InteractionResult): Promise<void>
  
  /** 获取互动统计 */
  getInteractionStats(): Promise<{
    totalInteractions: number;
    todayCount: number;
    mostFrequentType: InteractionType;
    lastInteractionTime: number;
  }>
}
```

### 6.3 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `foodId` | string | 是 | 食物标识符，映射到食物资源 |
| `toyId` | string | 是 | 玩具标识符，映射到游戏逻辑 |
| `commandId` | string | 是 | 训练指令标识，如 `"sit"` / `"shake_hand"` |
| `quantity` | number | 否 | 喂食数量，默认1，影响回复量 |
| `duration` | number | 否 | 游戏时长(ms)，越长精力消耗越多 |

## 7. MVP最小可用版本清单

### 7.1 Phase 1 MVP 核心功能

以下为**必须实现**的最小功能集，确保单端（Previewer/手机）可独立运行：

| 功能点 | 子项 | 优先级 | 说明 |
|--------|------|--------|------|
| 宠物精灵渲染 | idle/look_around/yawn/eat/sleep/pet_enjoy/happy_spin 动画 | **P0** | Canvas帧动画，至少6种动作 |
| 属性系统 | 六维属性初始化、读取、修改、自动衰减 | **P0** | 每10分钟自然衰减，RDB持久化 |
| 基础互动 | 抚摸(pet)、喂食(feed)、睡觉(sleep) | **P0** | 三种核心互动+属性反馈 |
| 情绪系统 | 基于属性值的情绪计算与转换 | **P0** | 6种情绪，属性驱动 |
