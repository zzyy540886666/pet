# 社交与系统模块 — 模块设计文档

> **模块名称**: 社交与系统模块 (Social & System Module)  
> **所属项目**: 鸿蒙虚拟数字宠物 (HarmoPet)  
> **模块版本**: v1.0  
> **更新日期**: 2026-04-15

---

## 1. 模块概述

### 1.1 模块定位

社交与系统模块包含宠物领养流程、成长档案管理、成就系统和应用设置等功能，是用户进入和使用 HarmoPet 的入口与辅助系统。这些功能增强用户的归属感和成就感，提升产品粘性。

### 1.2 模块子集

| 子模块 | 说明 | MVP优先级 |
|--------|------|----------|
| 宠物领养系统 | 首次使用引导：选择品种→命名→初始性格测试→领养完成 | **P0** |
| 宠物档案系统 | 基本信息/属性面板/性格报告/成长轨迹/互动统计/梦境日记/健康记录 | **P0** |
| 成就系统 | 多类别成就解锁，激励持续互动 | P1 |
| 设置中心 | 应用配置/AI服务地址/声音/通知/关于等 | **P0** |

### 1.3 依赖关系

```
社交与系统模块
├── 依赖: 核心宠物模块 (读取宠物属性/性格/记忆数据)
├── 依赖: 北向数据层 (RDB读写档案/成就数据)
├── 依赖: AI智能模块 (领养性格测试可能调用AI)
└── 被依赖: 所有页面 (领养页/档案页/设置页)
```

## 2. 宠物领养子系统

### 2.1 功能描述

新用户首次打开App时进入领养流程，引导用户选择宠物品种、命名、进行简单的性格倾向测试，最终完成领养并进入主界面。

#### 领养流程

```
┌────────┐   ┌────────┐   ┌────────┐   ┌────────┐   ┌────────┐
│ 欢迎    │──→│ 选择    │──→│ 命名    │──→│ 性格    │──→│ 领养    │
│ 页面    │   │ 品种    │   │ 宠物    │   │ 测试    │   │ 成功    │
│        │   │ 猫/狗/兔 │   │        │   │ 初始性格 │   │        │
└────────┘   └────────┘   └────────┘   └────────┘   └────────┘
 Step 1        Step 2       Step 3       Step 4       Step 5
```

#### 品种选择

| 品种 | ID | 性格倾向 | 特殊能力 | 声音风格 | 精灵资源 |
|------|-----|----------|----------|----------|----------|
| 小橘猫 | `cat` | 独立、慵懒 | 精力消耗慢 | 软萌喵叫 | cat_front/right/back/left.png |
| 柯基犬 | `dog` | 活泼、粘人 | 亲密度加成 | 元气汪汪 | dog_front/right/back/left.png |
| 小白兔 | `rabbit` | 温柔、安静 | 心情恢复快 | 轻柔吱吱 | rabbit_front/right/back/left.png |

#### 性格测试（Step 4）

通过3-5道趣味选择题确定宠物初始五维性格：

| 问题示例 | 选项A(影响维度) | 选项B(影响维度) |
|----------|-----------------|-----------------|
| "周末你喜欢怎么过？" | A.宅家看书 (+independence,-curiosity) | B.出门探险 (-independence,+curiosity) |
| "朋友难过时你会？" | A.默默陪伴 (+warmth) | B.想办法逗ta笑 (+cheerfulness,+naughtiness) |
| "看到新奇玩具？" | A.先观察研究 (+curiosity,-naughtiness) | B.马上玩起来 (-curiosity,+naughtiness) |

### 2.2 接口定义

```typescript
// 文件路径: services/AdoptionService.ets

export type PetBreed = 'cat' | 'dog' | 'rabbit';

export interface AdoptionProfile {
  name: string;                    // 宠物名字 (2-8字符)
  breed: PetBreed;                 // 品种
  adoptionDate: number;            // 领养时间戳
  initialPersonality: Record<PersonalityDimensionId, number>; // 初始性格
}

export class AdoptionService {
  private static instance: AdoptionService;
  
  public static getInstance(): AdoptionService
  
  // ---------- 领养流程 ----------
  /** 检查是否已完成领养（用于决定显示主页还是领养页） */
  isAdopted(): boolean
  
  /** 执行领养（保存宠物基本信息到RDB） */
  async adopt(profile: AdoptionProfile): Promise<boolean>
  
  /** 获取已领养的宠物基本信息 */
  getAdoptionProfile(): AdoptionProfile | null
  
  // ---------- 性格测试 ----------
  /** 获取性格测试问题 */
  getPersonalityQuizQuestions(): QuizQuestion[]
  
  /** 提交性格测试答案，计算初始性格 */
  calculateInitialPersonality(answers: Record<number, number>): Record<PersonalityDimensionId, number>
}

export interface QuizQuestion {
  id: number;
  question: string;
  options: Array<{
    label: string;
    personalityImpact: Partial<Record<PersonalityDimensionId, number>>;
  }>;
}
```

### 2.3 参数说明

| 参数名 | type | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `name` | string | **是** | - | 宠物名字，2-8个字符 |
| `breed` | PetBreed | **是** | - | 品种标识 |
| `initialPersonality.*` | number | 是(计算得出) | - | 各性格维度初始值 0~100 |

## 3. 宠物档案子系统

### 3.1 功能描述

档案系统以结构化方式展示宠物的全面信息，包括基本资料、六维属性趋势、性格分析报告、成长时间线、互动统计、梦境日记和健康记录。

#### 档案内容板块

| 板块 | 数据来源 | MVP优先级 | 内容说明 |
|------|----------|-----------|----------|
| 基本信息 | RDB: adoption_profile | **P0** | 名字/品种/出生日期/领养天数 |
| 属性面板 | PetAttributeService | **P0** | 六维属性实时数值 + 变化曲线图 |
| 性格报告 | PersonalityService | P1 | 五维雷达图 + 性格标签 + 演化趋势 |
| 成长轨迹 | RDB: pet_growth | P1 | 关键事件时间线(首次互动/阶段升级等) |
| 互动统计 | RDB: interaction_log | P1 | 总次数/今日次数/最常类型/活跃时段分布 |
| 梦境日记 | DreamService / RDB: dream_journal | **P0** | 梦境卡片列表(类型/内容/插图) |
| 健康记录 | AlertService + RDB: sensor_data | P1 | 预警历史 + 环境异常记录 |

### 3.2 接口定义

```typescript
// 文件路径: services/ProfileService.ets

export interface PetProfile {
  basicInfo: {
    name: string;
    breed: PetBreed;

  attributes: PetAttributes;      // 六维属性实时快照
  personality: {
    profile: PersonalityProfile;   // 性格详情

  stats: {
    totalInteractions: number;
    todayInteractions: number;

  };

}

export interface GrowthEvent {

}

export class ProfileService {
```

## 4. 成就子系统

### 4.1 功能描述

成就系统通过预设的多类成就目标激励用户持续与宠物互动，增加产品的游戏性和粘性。达成条件基于互动数据自动检测和解锁。

#### 成就类别与示例

| 类别 | 成就ID | 名称 | 解锁条件 | 奖励 | MVP |
|------|--------|------|----------|------|-----|
| **互动成就** | `first_meet` | 初次相遇 | 首次完成任意互动 | 亲密度+5 | **P0** |
| | `hundred_pats` | 百次之约 | 累计抚摸100次 | 解锁特殊动作 | P1 |
| | `thousand_chats` | 千语伙伴 | 累计对话1000条 | 特殊称号 | P2 |
| **成长成就** | `hatch_out` | 破壳而出 | 宠物度过初识期(>7天) | 解锁新皮肤 | **P0** |
| | `soul_mate` | 心有灵犀 | 亲密度达到80+ | 解锁专属梦境 | P1 |
| **探索成就** | `first_dream` | 第一个梦 | 宠物首次做梦 | 梦境相册解锁 | **P0** |
| | `voice_hour` | 话痨伙伴 | 语音累计1小时 | 音色解锁 | P1 |
| **特殊成就** | `through_thick_thin` | 患难与共 | 经历环境异常预警 | 勋章 | P1 |
| | `hundred_days` | 百日记 | 相处满100天 | 终极成就 | P2 |

#### 成就数据模型

```typescript
// 文件路径: model/AchievementModel.ets

export type AchievementCategory = 'interaction' | 'growth' | 'exploration' | 'special';

export interface AchievementDefinition {
  id: string;                        // 唯一标识
  name: string;                      // 成就名称
  description: string;               // 描述文案
  category: AchievementCategory;     // 所属类别
  icon: string;                      // 图标资源名
  condition: AchievementCondition;   // 解锁条件
  reward: AchievementReward;         // 奖励
  hidden: boolean;                   // 是否隐藏(未解锁前不可见)
  mvpPriority: 'P0' | 'P1' | 'P2';  // MVP优先级
}

export interface AchievementCondition {
  type: 'count' | 'threshold' | 'event' | 'duration';
  target: string;                    // 目标对象: interaction_type / attribute_id / event_name
  value: number;                     // 目标值
}

export interface AchievementReward {
  intimacyBonus?: number;            // 亲密度奖励
  unlockContent?: string;            // 解锁的内容ID(动作/皮肤/音色/称号)
  title?: string;                    // 称号
}

export interface AchievementState {
  achievementId: string;
  unlocked: boolean;                 // 是否已解锁
  unlockedAt?: number;               // 解锁时间戳
  progress: number;                  // 当前进度
  progressMax: number;               // 目标值(用于显示进度条)
}
```

### 4.2 接口定义

```typescript
// 文件路径: services/AchievementService.ets

export class AchievementService {
  private static instance: AchievementService;
  
  public static getInstance(): AchievementService
  
  // ---------- 生命周期 ----------
  /** 初始化成就系统，从RDB加载已解锁状态 */
  async initialize(): Promise<void>
  
  // ---------- 进度检查 ----------
  /**
   * 检查并尝试解锁成就（每次互动结束后调用）
   * @param eventType 触发的事件类型
   * @param context 附加上下文
   * @returns 本次新解锁的成就列表
   */
  checkAndUnlock(
    eventType: string, 
    context?: Record<string, any>
  ): Promise<AchievementDefinition[]>
  
  // ---------- 数据查询 ----------
  /** 获取所有成就及其解锁状态 */
  getAllAchievements(): AchievementState[]
  
  /** 获取已解锁成就列表 */
  getUnlockedAchievements(): AchievementState[]
  
  /** 获取未解锁成就列表 */
  getLockedAchievements(): AchievementState[]
  
  /** 获取按分类分组的成就 */
  getAchievementsByCategory(): Record<AchievementCategory, AchievementState[]>
  
  /** 监听新成就解锁事件 */
  onAchievementUnlocked(): Observable<AchievementDefinition>
}
```

### 4.3 参数说明

| 参数名 | type | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `eventType` | string | **是** | - | 事件类型，如 `"feed"` / `"chat"` / `"dream"` |
| `context` | object | 否 | `{}` | 附加上下文，可包含具体数值等信息 |

## 5. 设置中心子系统

### 5.1 功能描述

设置中心提供应用级别的配置管理能力，包括AI服务连接配置、声音设置、通知偏好、数据显示项、账号管理和关于信息等。

#### 设置项清单

| 分类 | 设置项 | 类型 | 默认值 | 说明 | MVP |
|------|--------|------|--------|------|-----|
| **AI服务** | API地址 | text | `http://localhost:8000` | FastAPI后端地址 | **P0** |
| | API Key | password | - | LLM API密钥(本地存储) | **P0** |
| | 自动重连 | switch | true | 断线后自动重试连接 | P1 |
| **声音** | 背景音乐 | switch | false | 是否播放BGM | P2 |
| | 音效音量 | slider | 0.8 | 互动音效音量 0~1 | P1 |
| | TTS音量 | slider | 0.9 | 语音回复音量 0~1 | P1 |
| **通知** | 属性提醒 | switch | true | 属性低于阈值时提醒 | **P0** |
| | 梦境通知 | switch | true | 宠物做梦后通知 | P1 |
| | 传感器预警 | switch | true | 南向异常预警 | P1 |
| **显示** | 语言 | select | zh-CN | 界面语言 | P2 |
| | 主题 | select | light | light/dark主题 | P1 |
| **数据** | 清除缓存 | button | - | 清除临时缓存 | P1 |
| | 导出数据 | button | - | 导出全部RDB数据 | P1 |
| | 重置宠物 | button | - | **危险操作**：重新领养 | P2 |
| **关于** | 版本信息 | readonly | v1.0.0 | 应用版本号 | **P0** |
| | 检查更新 | button | - | 检查是否有新版本 | P2 |
| | 项目信息 | readonly | - | 团队/赛题信息 | **P0** |

