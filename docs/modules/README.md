## MVP开发阶段对应关系

### Phase 1 — 核心骨架（第1周）

> 目标：Previewer/手机单端可运行的完整基础体验

| 涉及模块 | P0功能点 | 产出物 |
|----------|----------|--------|
| 01-核心宠物模块 | 宠物精灵动画(idle/eat/sleep/pet等6种动作)、六维属性+衰减、基础互动(喂食/抚摸/睡觉)、情绪系统 | PetSprite组件 + AttributeService + InteractionService |
| 05-社交系统 | 领养流程(5步)、档案基础信息页、设置-AI地址配置 | AdoptPage + SettingsPage |
| 06-北向架构 | 页面路由框架(MainPetPage/ChatPage/SettingsPage)、RDB初始化(8张表)、PersistentStorage | 完整页面骨架 + 数据持久化 |

**MVP验收标准**: Previewer中可以看到一只可互动的宠物，能喂食/抚摸/睡觉，属性会变化，数据不丢失。

### Phase 2 — AI赋能（第2周）

> 目标：宠物能对话、能语音交流、有记忆

| 涉及模块 | P0功能点 | 产出物 |
|----------|----------|--------|
| 02-AI智能模块 | LLM对话接入+Pet人格Prompt、Whisper STT、Edge-TTS TTS、VAD检测、预设回复池、文字情绪分析、做梦API | AI完整链路 |
| 07-AI后端 | FastAPI项目骨架、Chat API、Voice API(STT/TTS)、Emotion API、Dream API、Health Check | 可独立启动的AI服务 |

**MVP验收标准**: 能和宠物用文字或语音对话，宠物有记忆，能做梦。

### Phase 3 — 特色亮点（第3周）

> 目标：完善成长体系和社交玩法

| 涉及模块 | P0功能点 | 产出物 |
|----------|----------|--------|
| 05-社交系统 | 成就基础3个(first_meet/hatch_out/first_dream)、梦境日记页 | AchievementService + DreamsPage |
| 01-核心宠物模块 | 性格成长基础记录、更多动作动画 | PersonalityService |

### Phase 4 — 全息双端联动（第4-5周）

> 目标：硬件集成（可选增强）

| 涉及模块 | P0功能点 | 产出物 |
|----------|----------|--------|
| 03-IoT南向模块 | 星闪通信、温湿度采集、红外靠近/离开检测、蜂鸣器+LED控制、基础预警(L1) | Hi3863固件 + 北向南通信服务 |
| 04-全志双端模块 | Canvas 4方向渲染、WebSocket双端同步协议、状态消息定义 | HoloRenderer + HoloSyncService |

## 文档维护规范

1. **每个模块文档独立维护**，修改时仅更新对应的 `.md` 文件
2. **接口变更需同步更新**: 接口定义修改时，同时检查调用方文档是否需要同步更新
3. **MVP优先级标记**: 所有功能点标注了P0/P1/P2优先级，确保MVP范围清晰
4. **赛题合规标注**: 关键需求处标注了赛题对应关系，便于自查
