# 全息双端协同模块 — 模块设计文档

> **模块名称**: 全息双端协同模块 (Holographic Dual-Device Sync Module)  
> **所属项目**: 鸿蒙虚拟数字宠物 (HarmoPet)  
> **模块版本**: v1.0  
> **更新日期**: 2026-04-15

---

## 1. 模块概述

### 1.1 模块定位

全息双端协同模块是 HarmoPet 的**最大创新亮点**。通过 SC-3568HA 开发板 + 全息金字塔硬件方案，让虚拟宠物以**浮空3D形象**"活"在桌面上；手机作为交互控制端，通过 WiFi/WebSocket 与全息显示端实时同步，实现双端联动体验。

### 1.2 模块职责

| 职责 | 说明 |
|------|------|
| 全息渲染引擎 | Canvas 4方向画面合成 + 全息金字塔适配 |
| 双端同步协议 | 手机↔SC-3568HA 的实时状态/指令同步 |
| 双端角色分配 | 手机=控制端(交互)，全息端=展示端(呈现) |
| 降级兼容 | 无硬件时降级为单端 Previewer/手机运行 |

### 1.3 硬件组成

```
全息显示端硬件:
├── SC-3568HA 开发板 (5.5寸触摸屏, SwanLinkOS 5.0)
│   ├── 屏幕: 5.5寸 IPS, 分辨率 720×1280
│   ├── 触摸: 电容触摸屏（支持用户直接操作）
│   └── 系统: SwanLinkOS 5.0 (基于OpenHarmony)
├── 全息金字塔 (4片透明亚克力, 倒金字塔形)
│   ├── 尺寸: 底边~10cm, 高~8cm (适配5.5寸屏)
│   ├── 成本: 淘宝成品约 20-50 元
│   └── 放置: 金字塔底面贴合屏幕上方中心区域
└── 效果: 宠物看起来悬浮在空中, 约360°可见
```

> ⚠️ **赛题合规**：SC-3568HA 运行 SwanLinkOS 5.0（基于 OpenHarmony），符合赛题要求。无硬件时可使用 Previewer 展示。

## 3. 全息渲染子系统

### 3.1 功能描述

全息渲染引擎负责在 SC-3568HA 屏幕上绘制4个方向的宠物画面，通过全息金字塔反射形成浮空3D视觉效果。

#### Canvas绘制接口

```typescript
// 文件路径: components/holo-engine/HoloCanvas.ets

@Component
export struct HoloCanvas {
  /** 渲染引擎实例引用 */
  private renderer: HoloRenderer;
  
  /** Canvas组件的onDraw回调 */
  onDraw(context: CanvasRenderingContext2D): void {
    const state = this.renderer.getState();
    
    // 1. 绘制4方向宠物精灵
    for (const dir of ['front', 'right', 'back', 'left'] as ViewDirection[]) {
      this.drawPetInViewport(context, dir, state);
    }
    
    // 2. 绘制对话气泡（4方向各一份）
    if (state.showBubble) {
      for (const dir of ['front', 'right', 'back', 'left'] as ViewDirection[]) {
        this.drawBubble(context, dir, state.bubbleText);
      }
    }
    
    // 3. 绘制粒子特效
    for (const effect of state.activeEffects) {
      this.drawParticle(context, effect);
    }
    
    // 4. 绘制底部信息栏
    this.drawInfoBar(context, state.petAttributes);
  }
}
```

## 4. 双端同步协议

### 4.1 功能描述

双端同步协议定义了手机控制端和全息显示端之间的消息格式和通信机制，确保操作指令和状态数据实时一致。

#### 通信方式选型

| 方式 | 延迟 | 适用场景 | 推荐度 |
|------|------|----------|--------|
| HTTP REST | 50-200ms | 偶发性操作查询 | 低 |
| **WebSocket** | **10-50ms** | **实时状态同步（推荐）** | **高** |
| 分布式数据(@ohos.distributedData) | <10ms | 鸿蒙原生设备间同步 | 中(需多设备在线) |

MVP版本使用 **WebSocket** 作为主要通信方式。

#### 消息类型总览

```
═══════════════════════════════════════
  手机 → 全息端 (操作指令)
═══════════════════════════════════════
  type: "action"         → 用户操作(喂食/抚摸/玩耍等)
  type: "chat"           → 聊天消息(文字/语音)
  type: "state_request"  → 请求全量状态同步
  type: "config"         → 配置更新

═══════════════════════════════════════
  全息端 → 手机 (状态同步/事件通知)
═══════════════════════════════════════
  type: "state_sync"     → 宠物状态变化推送
  type: "sensor_event"   → 南向传感器事件转发
  type: "holo_event"     → 全息端特有事件(如触摸屏幕)

═══════════════════════════════════════
  双向 (心跳/控制)
═══════════════════════════════════════
  type: "ping" / "pong"  → 心跳保活
  type: "ack"             → 操作确认回复
```

### 4.2 接口定义

#### 手机 → 全息端：操作指令消息

```typescript
// ====== ActionMessage: 用户操作指令 ======

interface ActionMessage {
  type: 'action';
  timestamp: number;
  action: InteractionType;       // 'feed' | 'pet' | 'play' | 'train' | 'bath' | 'sleep'
  params?: {
    foodId?: string;              // 喂食时的食物ID
    toyId?: string;               // 玩耍时的玩具ID
    commandId?: string;            // 训练时的指令ID
  };
}

// ====== ChatMessage: 聊天消息 ======

interface ChatMessage {
  type: 'chat';
  timestamp: number;
  role: 'user' | 'pet';           // user=用户发消息, pet=宠物回复(含AI结果)
  content: string;                // 消息文本
  audioData?: string;             // TTS音频(Base64), 仅pet角色时有值
}

// ====== StateRequestMessage: 全量状态请求 ======

interface StateRequestMessage {
  type: 'state_request';
  timestamp: number;
  requestFullSync: boolean;       // true=需要完整状态, false=增量即可
}
```


## 6. 降级方案

### 6.1 无SC-3568HA硬件时的降级

| 场景 | 有全息端 | 无全息端（降级） | MVP处理方式 |
|------|---------|-----------------|------------|
| 宠物展示 | 全息浮空3D | **Previewer/手机屏幕2D** | 使用标准PetSprite组件替代HoloRenderer |
| 喂食互动 | 全息吃东西动画 | **手机屏幕吃东西动画** | 复用同一套动作逻辑，仅渲染目标不同 |
| 聊天回复 | 全息气泡+语音 | **手机气泡+语音** | UI完全相同，只是不在全息端显示 |
| 传感器联动 | 全息反应+手机通知 | **手机通知** | 直接接收南向事件（如果有的话），跳过全息端 |
| 双端同步 | WS实时同步 | **不需要同步（单端运行）** | HoloSyncService检测无对端则跳过 |

### 6.2 降级自动检测逻辑

```typescript
// 文件路径: services/HoloSyncService.ets

export class HoloSyncService {
  /**
   * 自动检测运行环境并选择合适的渲染模式
   */
  detectAndInitialize(): RenderMode {
    if (this.isHoloDevice()) {
      // 运行在全息设备上 → 启动全息渲染引擎
      this.holoRenderer = new HoloRenderer();
      this.startServer();  // 作为WS Server等待手机连接
      return 'holo_server';
    } 
    else if (await this.tryConnectHolo()) {
      // 手机端且能连上全息设备 → 作为WS Client连接
      return 'controller_client';
    } 
    else {
      // 纯单端模式 → 不启动任何WS服务
      return 'standalone';
    }
  }
  
  private isHoloDevice(): boolean {
    // 通过设备特征判断是否为SC-3568HA全息设备
    const deviceInfo = deviceinfo.getDefaultDisplaySync();
```
