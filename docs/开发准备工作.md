## 五、开发学习路线

### 5.1 北向学习路线（1-2周）

1. **ArkTS 语法基础**：TypeScript扩展语法、状态管理
2. **ArkUI 声明式开发**：组件体系、布局系统、动画框架
3. **数据管理**：PersistentStorage、AppStorage、关系型数据库
4. **网络与通信**：HTTP请求、WebSocket、Socket通信
5. **多媒体能力**：相机拍照、音频录制与播放、语音识别
6. **Canvas动画**：自定义绘制宠物精灵动画（**重点：4方向分屏渲染**）
7. **分布式能力**：设备发现、数据同步、迁移能力（**双端协同核心**）

### 5.2 全息渲染学习路线（额外1周）

1. **全息金字塔原理**：佩珀尔幻象（Pepper's Ghost）光学原理
2. **Canvas 4方向绘制**：同一画面分4象限渲染，每象限宠物朝不同方向
3. **动画帧调度**：4方向动画帧同步切换
4. **双端通信**：WebSocket消息协议设计
5. **全息适配**：屏幕亮度/对比度/画面布局调优

### 5.3 南向学习路线（1-2周）

1. **OpenHarmony轻量系统架构**：LiteOS-M内核、HDF驱动框架
2. **Hi3863开发基础**：GPIO/I2C/SPI/UART外设操作
3. **星闪（SLE）通信**：主从通信、数据传输协议
4. **传感器编程**：温湿度、加速度、红外等传感器数据采集
5. **南向北向数据桥接**：WiFi/星闪上报数据至北向应用

### 5.4 AI能力学习路线（1周）

1. **LLM接入**：DeepSeek API调用、Prompt工程、角色设定
2. **语音交互**：Whisper部署、Edge-TTS集成、语音活动检测
3. **情绪识别**：摄像头+表情分析、情感计算基础
4. **记忆系统**：向量嵌入、ChromaDB存储与检索

## 九、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| ⚠️ OpenHarmony合规风险 | 作品不符合赛题要求 | 确保基于OpenHarmony/SwanLinkOS开发，不使用HarmonyOS NEXT |
| 硬件开发板获取不确定 | 无全息/南向增强 | **不影响核心功能**，Previewer完整演示；硬件到位后快速集成 |
| SC-3568HA获取困难 | 无全息展示 | Previewer兜底，现场视频辅助展示全息效果 |
| 全息金字塔效果差 | 视觉体验不佳 | 购买成品金字塔+调优屏幕参数；备用视频展示最佳效果 |
| 双端同步延迟 | 交互不流畅 | 同步状态指令而非帧数据，全息端预缓存动画 |
| AI服务延迟 | 交互体验差 | 本地缓存+预设回复+异步处理，RDB保证数据不丢失 |
| 星闪通信不稳定 | 南北向联动失败 | 备选WiFi通信方案 |
| 开发周期紧张 | 功能不完整 | MVP优先：手机端核心→全息端渲染→AI增强→南向扩展 |
