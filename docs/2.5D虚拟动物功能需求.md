# Pet2.0 - 2.5D虚拟宠物功能需求文档

## 1. 业务背景
由于 HarmonyOS 模拟器环境以及部分机型对 `Component3D` 和 `SceneView` 的 OpenGL ES / Vulkan 硬件渲染支持不完善（易导致黑屏、着色器缺失），现将原有真 3D 渲染方案降级重构为 **2.5D（伪 3D）互动渲染方案**。

在保证极佳性能兼容性的前提下，利用 2.5D 技术依然能带给用户栩栩如生、具有空间感和物理反馈的宠物互动体验。

## 2. 核心功能需求

### 2.1 2.5D 视效表现
*   **高帧率流畅动画**：使用 Lottie 矢量骨骼动画或高质量预渲染帧序列（Frame Animation）替代原生 3D 骨骼驱动。要求提供待机（Idle）、开心（Happy）、进食（Eating）等状态的无缝切换。
*   **视差空间感（Parallax Effect）**：通过分层技术（UI 背景层、宠物阴影层、宠物主体层、UI 前景气泡层），在用户设备倾斜（基于陀螺仪 Sensor）或滑动屏幕时，各层产生不同速率的相对位移，营造强烈的景深和空间感。
*   **拟真光影**：光影随环境和动画本身提前烘焙或通过 UI 阴影（`shadow`）进行动态叠加，保持 UI Design System 规定的深色沉浸感。

### 2.2 交互反馈（Interaction）
*   **滑动跟随（PanGesture）**：
    *   用户在屏幕上左右滑动时，宠物本体会有基于 `.rotate({x,y,z})` 的 2.5D 矩阵微转反馈，模拟视角转动（限制在安全角度内防止穿帮）。
    *   松手后，通过弹簧动画（Spring Curve）自然回正。
*   **捏合缩放（PinchGesture）**：
    *   支持双指放大/缩小宠物查看细节，并附带弹性阻尼效果。
*   **触摸响应（Touch/Click）**：
    *   点击宠物本体的特定部位（如抚摸头部），触发专属互动动画（如开心），并伴随缩放物理反馈（`.scale({ x: 0.95, y: 0.95 })`）。

### 2.3 性能与兼容性
*   **极速首屏**：杜绝 3D 引擎初始化带来的加载黑屏，实现页面秒开。
*   **全机型兼容**：不依赖 GPU 高级着色器，能在所有 HarmonyOS NEXT 模拟器和真机上流畅运行（API 12+）。
*   **低功耗**：避免后台 3D 渲染常驻导致发热，动画静止时停止 UI 重绘。

## 3. 页面模块更新
*   **首页 (Index.ets)**：移除 `SceneView`，接入 2.5D 宠物精灵组件 `PetSprite.ets`。
*   **宠物档案 (PetProfilePage.ets)**：更新为空状态及 2.5D 档案卡片的展示逻辑。
*   **数据模型 (PetModel.ets)**：移除 `modelPath` (glb)，增加 `lottiePath` (json) 或 `spriteSheet` (预渲染帧图) 字段。
