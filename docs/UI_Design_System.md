# Pet2.0 UI 设计规范 (UI Design System)

本文档基于 Figma 设计稿提取，总结了 Pet2.0 应用中通用的 UI 配色、组件设计、风格及布局规范，以确保整个 HarmonyOS NEXT 应用的视觉与交互一致性。

## 1. 配色方案 (Color Palette)

应用整体采用**深色模式 (Dark Theme)** 为主的沉浸式视觉风格，辅以明亮的品牌色作为点缀和交互引导。

*   **页面背景色 (Background)**
    *   主背景色：深海蓝/暗青色（如 `#081E2A`, `#012A3E`）。为页面提供深邃的沉浸感。
*   **卡片与容器底色 (Surface/Card)**
    *   深灰蓝（如 `#2A3641`, `#31404D` 或带有透明度的黑色）。用于区分内容层级，承载列表项或网格项。
*   **主色调/强调色 (Primary/Accent)**
    *   明亮蓝/青色（如 `#0894BD`, `#1DA1F2`）。用于主操作按钮（Primary Button）、选中状态边框、以及关键数据的强调（如积分按钮）。
*   **文本颜色 (Typography Colors)**
    *   主要文本 (Primary Text)：纯白 `#FFFFFF`，用于大标题、卡片主内容。
    *   次要文本 (Secondary Text)：浅灰色或带透明度的白色（如 `rgba(255,255,255,0.7)`），用于描述、副标题或未选中状态。

## 2. 组件设计规范 (Component Design)

*   **按钮 (Buttons)**
    *   **主按钮 (Primary)**：背景色为品牌蓝（`#0894BD`），纯白文字，大圆角（`borderRadius(16)` 或胶囊形状）。
    *   **次要/标签按钮 (Secondary/Tags)**：深色背景，带有品牌蓝描边（Border），或者毛玻璃/半透明效果（如 Onboarding 页面的浮动标签）。
    *   **右上角操作按钮**：通常为胶囊形，包含图标和文字（如“积分”、“当前宠物”），背景色通常为蓝色或深灰色。
*   **卡片与列表项 (Cards & List Items)**
    *   **圆角**：统一使用较大圆角，推荐 `borderRadius(16)`。
    *   **边框状态**：选中的卡片会有一圈高亮的蓝色边框（Border），未选中时无边框或暗色边框。
    *   **阴影 (Shadow)**：轻微的阴影或内部渐变，用于凸显卡片的立体感和层次感。
*   **顶部导航栏 (Top Navigation)**
    *   左侧统一为返回图标（带有点击缩放动效）。
    *   左侧/居中显示页面标题及副标题（如 `Pet Profile \n Feed Me`）。
    *   右侧预留操作区（如显示积分或切换宠物）。

## 3. 布局与间距 (Layout & Spacing)

遵循 HarmonyOS 栅格与间距规范，采用 8vp 为基准的设计：

*   **页面边距 (Page Margin)**
    *   左右安全边距通常为 `16vp` 或 `24vp`。
*   **组件间距 (Spacing)**
    *   紧凑信息间距：`8vp`。
    *   常规内容区块间距：`16vp`。
    *   大区块/模块间距：`24vp` 或 `32vp`。
*   **典型排版模式**
    *   **列表模式 (Column)**：用于 Personal Care 页面，纵向排列的圆角卡片，内部采用左右结构（左图标+右文字）。
    *   **网格模式 (Grid)**：用于 Feed Me 页面，两列布局，卡片内元素居中对称（顶部图片+底部文字）。

## 4. 视觉风格与动效 (Style & Animation)

*   **图片与插画**
    *   大量使用去背景的高清 3D 渲染图或高质量实拍图（如宠物、食物、人物）。
    *   图片通常配有圆形底板或光晕效果作为背景衬托。
*   **装饰元素**
    *   背景中偶尔出现悬浮的几何图形（圆点、光斑）以增加页面的活泼感。
*   **交互动效 (Interaction)**
    *   **点击反馈**：所有可点击元素（按钮、卡片、返回图标）必须配置缩放动效（`.scale` 结合 `animation`），增强交互的物理手感。
    *   **状态切换**：颜色和边框的切换需平滑过渡。

## 5. 开发约束 (Development Constraints)

根据 `open.md` 规范要求：
1.  **纯中文界面**：Figma 中的英文需在实际开发中翻译为准确的中文。
2.  **极简图标**：不使用 Emoji，统一采用 SVG 矢量图标。
3.  **严格模式**：ArkTS 遵循严格类型，禁用 `any`，正确使用状态装饰器。
